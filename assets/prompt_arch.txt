This repository demonstrates when late chunking helps or hurts retrieval quality. It compares standard chunk embedding against late chunking using Jina.ai embeddings, Elastic/Elasticsearch vector search, and Elastic rank evaluation metrics.

Core message to communicate:
Late chunking is a specialized retrieval technique, not a universal upgrade. It degrades retrieval for self-contained Product FAQ entries because context bleeds between similar neighboring chunks. It improves retrieval for cross-referential SEC 10-K filing sections because document-level context resolves defined terms and references.
